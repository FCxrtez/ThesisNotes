## alineacion de mantisas

Cuando hago la laineacion en el system velrilog primero la shifteo `(packed_mantissa[i] << 3 * PRECISION_BITS + 4)` y luego `>> shifts[i]`. El ultimo no deberia ser de `shifts[i] + 3` pues queremos dejar el MSB para el signo y los siguientes dos bits para ovf? En modelo de python no tenemos ese problema porque tenemos precision infinita.

La otra opcion seria tratar `aligned_matissas` como la alineacion nomas, y luego metiendo estas mantissas en un bus con 3 bits mas a izq, seria mas prolijo. En este caso el primer shifteo deberia ser `(packed_mantissa[i] << 3 * PRECISION_BITS)`
