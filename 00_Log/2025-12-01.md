# Chipyard

[Video](https://www.youtube.com/watch?v=EXbs5VSv19c) de conferencia chipyard/firesim donde introducen el [framework](https://chipyard.readthedocs.io)

Detalles:

+ CVA6 usa el estandar de interfaz AXI4
+ Spike i an Open-Source RISC-V ISA simulator. Fast, extensible, c++ functional model. simil QEMU
+ protocolo del sistema de memoria "coherent interconnect, the high level protocol is tile link"
+ Existen Drop-in Verilog integrations para cva6 AMBA-to-TileLink que comunica el protocolo de comunicacion de CVA6 (AXI4) con TileLink
+ este protocolo sirve para comunicarse con sistema de memoria o sistema externo al core en general? Creo que con el SystemBus en general
+ Custom SoC Configuration -> RTL Generators -> RTL Build Process
+ Similar al protecto de CVA6, la configuracion del SoC esta definida por una clase Config
+ Configuracion es tomada por los RTL Generators
+ 3 niveles de configuracion:  
  + Digital system
  + Chip IO
  + Harness (se refiere a la plataforma. RTL simulation, FPGA, etc)
+ En min 48 se explica que ese es el punto donde se genera el .sv que luego podriamos tomar para simular de la manera que queramos. Es el punto mas bajo desde donde pordemos tomar algo usable de chipyard
