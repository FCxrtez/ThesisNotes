# Sobre la configuracion de FPNew, Proceso de compilacion? Como modificar previo a sintesis?

En el [github](https://github.com/openhwgroup/cvfpu/blob/2c7947726a0859c08cc27cbbd45662424dedb41e/docs/README.md#top-level-interface) de FPNew podemos encontrar la documentacion de arquitectura, configuracion e interfaz top-level.

En la seccion de Arquitectura hay una sexxion que explica que se puede configurar la cantidad de registros que tienen los pipelines de las unidades mas finas del calculo. Para tesis buscar tweakear estos valores para buscar el punto de mayor performance como un punto posible para un analisis mas profundo. Se podra hacer un cambio arrquitectural mediante system verilog?
Todavia no lei alguna seccion que comente la replicacion de slices de un mismo formato dentro de un grupo de operaciones. Capaz este cambio debamos introducirlo nosotros. Buscar manera de conseguir el codigo del procesador "compilado". Me imagino que esa es una etapa en donde podemos buscar realizar el cambio. Puede que nos limite la interfaz diseñada por los creadores.

**Efectivamente**, podemos alterar las stages del pipeline a traves del paquete de configuracion con el tipo **fpu_implementation_t** 
```sv
typedef struct packed {
  opgrp_fmt_unsigned_t   PipeRegs;
  opgrp_fmt_unit_types_t UnitTypes;
  pipe_config_t          PipeConfig;
} fpu_implementation_t;
``` 

Esto es controlado por PipeRegs, por default esta seteado a cero es decir que es un circuito puramente combinacional.
